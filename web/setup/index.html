Welcome to Cassiel Drive Setup

This wizard will guide you through creating Google Cloud OAuth credentials so Cassiel Drive can securely access your Google Drive accounts.

⏱
Takes about 5 minutes

You'll need a Google account and a web browser. No coding required.

🔒
Your credentials stay private

Your Client ID and Secret are entered only in your app. This wizard runs entirely in your browser — nothing is sent to any server.

☁️

Step 1: Create a Google Cloud Project

First, we need a Google Cloud project to hold your OAuth credentials.

1

Click the button below to open the Google Cloud Console:

Open Google Cloud Console ↗
2

Set the Project name to:

Cassiel Drive
3

Leave Organization as "No organization" and click Create.

4

Wait a few seconds for the project to be created, then make sure it's selected in the top dropdown.

⚡

Step 2: Enable Google Drive API

Enable the APIs that Cassiel Drive needs to access your files.

1

Click below to enable the Google Drive API:

Enable Google Drive API ↗
2

Click the blue "Enable" button on the page that opens.

3

(Optional) Also enable the People API for profile pictures:

Enable People API ↗
📋

Step 3: Configure OAuth Consent Screen

This screen tells Google what your app is called and who can use it.

1

Open the OAuth consent screen page:

Open Consent Screen ↗
2

Select "External" as the User Type, then click Create.

3

Fill in these fields:

App name
Cassiel Drive
User support emailYour Gmail address
Developer emailYour Gmail address
4

Click Save and Continue through the remaining steps (Scopes, Test Users). We'll add test users next.

👥

Step 4: Add Test Users

While your app is in "Testing" mode, only emails listed here can sign in.

⚠️
This is why "Access Blocked" happens!

If the Gmail you want to sign in with is NOT listed as a test user, Google will block access. Add ALL Gmail accounts you plan to use.

1

Go to the Test Users section:

Open Consent Screen ↗

Scroll down to "Test users" section, or click on the "Test users" tab.

2

Click "+ ADD USERS" and enter the Gmail addresses you want to use with Cassiel Drive. Add each one on a new line.

3

Click Save. Wait 2-5 minutes for changes to take effect.

🔍 Validate Your Email

Enter the email you'll sign in with to check the format:

🔑

Step 5: Create OAuth Credentials

Now create the Client ID and Secret that Cassiel Drive will use.

🚨
CRITICAL: Choose "Desktop app" type!

If you select "Web application" or "Android", your app will get "Access Blocked". The Desktop app type is required for Cassiel Drive's authentication flow.

1

Open the Credentials page:

Open Credentials Page ↗
2

Click "+ CREATE CREDENTIALS" → select "OAuth client ID".

3

Set Application type to:

✓ CORRECT
Desktop app

Allows loopback redirect

✗ WRONG
Web application

Will cause "Access Blocked"

✗ WRONG
Android

Uses different flow

4

Set the name to:

Cassiel Drive Desktop
5

Click Create. A dialog will show your Client ID and Client Secret.

✅

Step 6: Enter Your Credentials

Paste your Client ID and Secret below to validate them. These stay in your browser — nothing is sent to any server.

📱 Scan to Import on Mobile

Open Cassiel Drive on your phone → Settings → Scan QR Code

Or manually paste the Client ID and Secret in the app's Settings screen.

🎉
